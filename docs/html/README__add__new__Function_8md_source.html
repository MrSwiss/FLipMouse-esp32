1 # Howto add a new function
2 
3 If you want to add a new function to the FLipMouse as an independent functional task, you have to take care of following steps:
4 
5 ## Implement a function task
6 
7 Create a file in main/function_tasks with a "task_" prefix, with at least following methods:
8 
9 * task_<name> A task with your functionality. Please note that each functional task should take care of a virtual button set to VB_SINGLESHOT. In this case
10 the task is NOT allowed to block the caller, because there should be triggered an action just-in-time without any triggered virtual buttons.
11 * task_<name>_getAT This method is used to reverse parse an AT command to an original string, which can be sent back to the host for showing the current configuration.
12 
13 The parameters for your function task need to have at least an uint8_t field for the virtual button number.
14 
15 ## Add a nwe command type and config
16 
17 If necessary add your new type of function to the command_type_t enum. This enum is used to group functions. For example, all mouse related commands and actions are covered by "T_MOUSE".
18 In addition, if you want to store config settings, add it to the generalConfig_t struct. Everything in this struct is stored as general settings for this slot. The current config can
19 be obtained by configGetCurrent() (include config_switcher.h for this).
20 
21 ## Add files to config switcher
22 
23 For a task to be loaded on a config switch, it needs to be known to the config_switcher.c file, which is used to reload a config by attaching tasks to virtual buttons (if and only if
24 the virtual button is NOT VB_SINGLESHOT):
25 
26 * Add your header file to config_switcher.h
27 * In configSwitcherTask, add your command type to switch which is used to determine the parameters to be allocated on a task switch (around line 300-360).
28 
29 
30 ## Add command to command parser
31 
32 The serial command parsing from the host's commands (based on an AT command set) is done in task_commands.c.
33 Command parsing is split into smaller portions grouped into functions, named do<name>Parsing().
34 
35 Either you place your new command into one of the existing methods OR you create a new method and add it to the task_commands, which calls each parser.
36 
37 Do following steps to enable parsing of your new command:
38 
39 * Add a new parameter memory allocate to the beginning of task_commands. In addition, add a check if the pointer is valid (directly below)!
40 * Add your do<Name>Parsing method to the list of all parsers, around line 1400.
41 
42 A few additional infos:
43 
44 * Each do<Name>Parsing method needs to return a value of type parserstate_t. Please consult the source documentation for information which value to return.
45 * Don't forget to clear the parameter memory before doing anything with it.
46 * To detect a command, you can use the macros CMD (which compares a full AT command like "AT RO") or CMD4 (used to compare 4 characters like "AT M").
