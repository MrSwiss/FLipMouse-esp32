This method parses a 16bit unicode character to a keycode by simply invoking parse_for_keycode two times. Some parameter in the FLipMouse/FABI configuration might be using 16bits for characters, so in this case this function is used.

This method parses one incoming byte for the given locale. It returns 0 if there is no keycode or another byte is needed (Unicode input). If a modifier is needed the given modifier byte is updated.

In the case of a necessary deadkey sequence, the second keystroke is returned and the first keystroke is written to the given parameter.
